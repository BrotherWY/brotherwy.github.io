搭建一个react项目

Aug 15, 2017 | 技术 | 阅读

公司的weex android app做完了，自己也在学着react,正愁不知道拿什么练手。公司的这个app数据是现成的，而且设计图也是有的，正好拿来写一个webapp。哈哈哈。。机智如我！！！

技术栈

  • react
  • axios 网络请求
  • less 预处理器
  • react-router 路由
  • react-redux 状态管理
  • node 数据转发

webpack配置

配置生产环境和开发环境:

  • production 生产环境
  • development 开发环境

在配置生产环境和开发环境时，利用的是process.env.NODE_ENV这个参数（process是node的全局对象），因为我是windows平台，所以设置process.env.NODE_ENV用的windows脚本，然后在你要判断的地方调用process.env.NODE_ENV 就可以获取值，默认是development

mac或者linux直接在启动命令中加入:export NODE_ENV=production

webpack里的配置

提取公共代码:

在项目中有一些代码或者模块是固定的，如果不对他进行处理，每次请求的时候都去请求一次，无疑是对性能的巨大损耗，所以我们把这些代码提取到一个公共文件中，这时浏览器会对其进行缓存，但是这仅仅只是协商缓存(304) 浏览器还是会去和服务器通信去验证该文件是否更新，我们完全可以省略这个步骤从而提高性能，让浏览器对其进行强制缓存，不再让它去验证，那么我们如何知道这个文件有没有更新呢，很简单，在抽取代码时加入hash值，eg:index.js?v=12s3421。 我们在更新的时候也不再进行覆盖式更新，这样问题就解决了。
